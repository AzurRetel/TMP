Filebeat Windows
# Ir a carpeta de trabajo
cd C:\Temp

# Descargar Filebeat
Invoke-WebRequest `
-Uri "https://artifacts.elastic.co/downloads/beats/filebeat/filebeat-9.4.1-windows-x86_64.zip" `
-OutFile "filebeat-9.4.1-windows-x86_64.zip"

# Crear directorio destino
New-Item -ItemType Directory -Path C:\Elastic -Force

# Descomprimir
Expand-Archive `
-Path .\filebeat-9.4.1-windows-x86_64.zip `
-DestinationPath C:\Elastic `
-Force

# Entrar a Filebeat
cd C:\Elastic\filebeat-9.4.1-windows-x86_64

# Editar configuración
notepad .\filebeat.yml

# Validar sintaxis
.\filebeat.exe test config -e

# Validar conectividad a Elasticsearch
.\filebeat.exe test output -e

# Instalar servicio
.\filebeat.exe install service

# Iniciar servicio
Start-Service filebeat

# Verificar estado
Get-Service filebeat

# Ver logs del servicio
Get-WinEvent -LogName Application -MaxEvents 50

  --fleet-server-insecure-http \

sudo systemctl enable elastic-agent  # Arranque automático

6. Ejecución de aplicación Java con Elastic APM
El Elastic APM Agent se integra en la JVM de la aplicación Java mediante -javaagent, lo que permite la instrumentación automática sin necesidad de modificar el código.
Existen dos formas principales de habilitarlo, dependiendo del entorno de ejecución.

Ejecución manual (modo JAR / pruebas / desarrollo)
Este método se utiliza cuando la aplicación se ejecuta directamente con java -jar.

Comando:
java -javaagent:/opt/elastic/apm/elastic-apm-agent.jar \
 -jar aplicacion.jar

Descripción:
* java: ejecuta la máquina virtual de Java
* -javaagent: carga el agente de Elastic APM dentro de la JVM
* aplicacion.jar: aplicación Java que será instrumentada
En este modo, el agente se activa únicamente mientras la aplicación está en ejecución.
Ejecución como servicio en Linux
Este método se utiliza en entornos productivos donde la aplicación se ejecuta como servicio del sistema.

Opción A: systemd
Editar el servicio:
sudo vim /etc/systemd/system/app.service
Ejemplo de configuración:
[Service]
ExecStart=/usr/bin/java \
 -javaagent:/opt/elastic/apm/elastic-apm-agent.jar \
 -jar /opt/app/aplicacion.jar

Aplicar cambios:
sudo systemctl daemon-reload
sudo systemctl restart app

Opción B: JVM options (entornos que lo soportan)
En algunos entornos, la JVM puede leer parámetros desde archivos de configuración.
Ubicaciones típicas:
/etc/jvm.options
o
/etc/jvm.options.d/apm.options

# Contenido del archivo:
-javaagent:/opt/elastic/apm/elastic-apm-agent.jar

En este caso, el agente se carga automáticamente al iniciar la JVM.
Diferencias principales
Método
	Uso típico
	Gestión
	java -jar con -javaagent
	desarrollo / pruebas
	manual
	systemd o jvm.options
	producción
	sistema operativo
	Resumen
El agente de Elastic APM puede activarse de dos formas:

1. Incluyéndolo directamente en el comando de ejecución de Java.

2. Integrándolo en la configuración del sistema o de la JVM en entornos productivos.
En ambos casos, el objetivo es el mismo: instrumentar la aplicación Java para enviar métricas, trazas y errores a Elastic APM sin modificar el código fuente.
